# Phase 3: React Prototype Export

## Goal
Add a "Create Prototype" button that generates an interactive React + Tailwind web app from the Figma frames designed during the conversation. The prototype auto-opens in the designer's browser.

## Prerequisites
- Phase 2 complete: Claude can create/modify designs using library components
- Node.js installed on designer's MacBook (for running Vite dev server)

## Architecture for This Phase

```
Designer clicks "Create Prototype"
    │
    ▼
Plugin tracks which frames were created/modified
    │
    ▼
Plugin Sandbox exports:
  - Node tree data (JSON) for each frame
  - Rendered PNG images of each frame
    │
    ▼
Plugin UI sends to Cloudflare Worker:
  - Frame data + images
  - Conversation history (for intent context)
    │
    ▼
Claude generates React + Tailwind code:
  - App.tsx with routing between screens
  - Page components for each frame
  - Working form inputs, dropdowns, modals
  - Hover/focus states, transitions
    │
    ▼
Plugin writes files to local filesystem (via proxy or download)
    │
    ▼
Vite dev server starts → browser opens → interactive prototype
```

## What to Build

### 1. Frame Tracker (`plugin/src/ui/hooks/useFrameTracker.ts`)

Track frames created or modified during the conversation:
- Listen for tool execution results (create_frame, update_node, etc.)
- Maintain a list of frame node IDs that were touched
- When "Create Prototype" is clicked, use this list to know which frames to export

### 2. Export Tools (`plugin/src/tools/export.ts`)

Add sandbox handlers to export design data:
- `export_frame_data(frameId)` — Serialize the full node tree of a frame as JSON (types, positions, sizes, fills, text content, component instances, auto-layout properties)
- `export_frame_image(frameId)` — Render the frame as PNG using `node.exportAsync({ format: "PNG", constraint: { type: "SCALE", value: 2 } })`
- `export_all_frames()` — Export all tracked frames in one batch

### 3. Code Generation Prompt (`plugin/src/codegen/prompt.ts`)

Build the Claude prompt that converts Figma data to React code:
- System prompt with React + Tailwind code generation instructions
- Include the frame JSON data and conversation history
- Include exported PNGs as base64 images (visual reference for Claude)
- Request specific output format: one file per page, plus App.tsx with routing
- Include instructions for interactive elements (forms, navigation, modals)

### 4. Project Scaffold (`plugin/src/codegen/scaffold.ts`)

Generate a complete Vite + React + Tailwind project:
- `package.json` with dependencies (react, react-dom, react-router-dom, tailwindcss)
- `vite.config.ts`
- `tailwind.config.js`
- `index.html`
- `src/main.tsx` (entry point)
- `src/App.tsx` (routing between pages — generated by Claude)
- `src/pages/*.tsx` (one per frame — generated by Claude)
- `src/components/*.tsx` (shared components — generated by Claude)

### 5. Prototype Panel (`plugin/src/ui/components/PrototypePanel.tsx`)

UI for the "Create Prototype" flow:
- "Create Prototype" button (shown when frames have been created)
- Progress indicator during generation (exporting frames... generating code... starting server...)
- Preview URL display + "Open in Browser" link when ready
- Error handling and retry option

### 6. Output Delivery

Two options for getting the generated code to the designer's machine:

**Option A (PoC — simpler):** Generate code, let designer download as a zip file from the plugin, then `npm install && npm run dev` manually.

**Option B (better UX):** Proxy writes files to a temp directory, runs `npm install && npx vite --open` via a small CLI companion tool the designer has installed.

Start with Option A for the PoC.

## Key Constraints

- **Plugin UI can't write to filesystem**: The iframe can only trigger a download. For the PoC, generate a zip file and offer it as a download.
- **Image export is async**: `exportAsync` returns a `Uint8Array` — convert to base64 for sending to Claude.
- **Token limits**: A complex multi-frame design may exceed Claude's context. Mitigate by:
  - Summarizing frame data rather than sending raw JSON
  - Sending one frame at a time if needed
  - Using the conversation history to provide intent context instead of raw data
- **Code quality**: Claude generates the code in one shot. It won't be perfect. This is a prototype, not production code.
